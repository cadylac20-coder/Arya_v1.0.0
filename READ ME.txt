# UMMKOV v1.0.0 — Inquiry and Transactional Bot

This is a production-ready AI and transaction management system that handles:
- **AI Chat** (understand needs, recommend packages)
- **Bookings** (create, hold, cancel)
- **Ancillaries** (insurance, visas, upgrades)
- **Payments** (generate links, track)
- **Support** (custom requests, agent handoff)

## API Endpoints

### Chat Endpoints

**POST /chat** — Talk to AI
```bash
curl -X POST http://localhost:8000/chat \
  -H "X-API-Key: mkov-dev-key-2026" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "I want to go to Bali",
    "session_id": "optional-id"
  }'
'''

Response:
```json
{
  "reply": "That's wonderful! Bali is stunning...",
  "session_id": "abc123",
  "suggested_actions": [
    {
      "action": "hold_booking",
      "label": "Hold This Package"
    }
  ]
}
```

### Action Endpoints

**POST /menu/{session_id}** — Show quick action menu
```bash
curl -X POST http://localhost:8000/menu/abc123 \
  -H "X-API-Key: mkov-dev-key-2026"
```

Response:
```json
{
  "session_id": "abc123",
  "greeting": "Welcome Travel Partner! 👋",
  "quick_actions": [
    {
      "id": "get_confirmation",
      "label": "📋 Get Booking Confirmation",
      "action": "get_confirmation"
    },
    {
      "id": "hold_booking",
      "label": "🔒 Hold / Reserve Package",
      "action": "hold_booking"
    },
    ...
  ]
}
```

**POST /action** — Execute action
```bash
curl -X POST http://localhost:8000/action \
  -H "X-API-Key: mkov-dev-key-2026" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "hold_booking",
    "session_id": "abc123",
    "details": {
      "destination": "Bali",
      "travel_dates": "Feb 15-25",
      "duration_hours": 24
    }
  }'
```

Response:
```json
{
  "status": "success",
  "action": "hold_booking",
  "message": "🔒 BOOKING HELD\n\nHold ID: HOLD-ABC123\nDestination: Bali\nHold Valid For: 24 hours",
  "data": {
    "hold_id": "HOLD-ABC123",
    "hold_expiry_hours": 24
  }
}
```

## Supported Actions

### 1. Get Booking Confirmation
Retrieve existing booking details.
```json
{
  "action": "get_confirmation",
  "session_id": "abc123",
  "booking_id": "BK-001"
}
```

### 2. Hold Booking (Reserve 24 hours)
Reserve a package without payment.
```json
{
  "action": "hold_booking",
  "session_id": "abc123",
  "details": {
    "destination": "Bali",
    "travel_dates": "Feb 15-25",
    "duration_hours": 24
  }
}
```

### 3. Cancel Booking
Process cancellation and calculate refund.
```json
{
  "action": "cancel_booking",
  "session_id": "abc123",
  "booking_id": "BK-001",
  "details": {
    "reason": "Emergency work"
  }
}
```

### 4. Add Ancillary
Add travel insurance, visa, upgrades, etc.
```json
{
  "action": "add_ancillary",
  "session_id": "abc123",
  "booking_id": "BK-001",
  "details": {
    "type": "travel_insurance",  // or visa_assistance, hotel_upgrade, etc.
    "details": {"coverage": "full"}
  }
}
```

### 5. Generate Payment Link
Create payment link for booking.
```json
{
  "action": "payment_link",
  "session_id": "abc123",
  "details": {
    "booking_id": "BK-001"
  }
}
```

### 6. Custom Request
Create support ticket for complex requests.
```json
{
  "action": "custom_request",
  "session_id": "abc123",
  "details": {
    "request_text": "Group of 50 people, need corporate package"
  }
}
```

## Setup & Run

### Install
```bash
pip install -r requirements.txt
```

### Configure
```bash
cp .env.example .env
# Edit .env with your OpenAI API key
```

### Run
```bash
uvicorn main:app --reload
```

Visit: `http://localhost:8000/docs` for interactive API testing

### Test
```bash
python example_flow.py
```

Shows 4 complete customer journeys:
1. Inquiry → Hold → Payment
2. Check confirmation → Add ancillaries
3. Cancellation → Refund
4. Custom request → Support ticket

## Database Schema

10 tables track everything:

```
conversations      — All chat history
api_keys          — Authentication
leads             — Lead information
bookings          — Confirmed bookings
holds             — 24-hour reservations
ancillaries       — Add-ons (insurance, upgrades)
payments          — Payment tracking
support_tickets   — Custom requests
extracted_data    — Structured lead info
```
